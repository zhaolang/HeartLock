## 1. Purpose（目的）

本文档是 HeartLock（心锁）的产品需求说明书（PRD），旨在完整定义产品的核心理念、功能范围、用户角色、功能需求和非功能需求，为后续设计、开发、测试提供统一的产品级参考依据。

本文档适用于产品经理、设计师、后端开发工程师、HarmonyOS 客户端开发工程师、测试工程师及项目管理人员。

---

## 2. Scope（范围）

### 2.1 产品范围

HeartLock（心锁）是一款基于 HarmonyOS NEXT 的情感匹配应用。用户可以在完全保密的情况下，将一份"喜欢"放进心锁。只有当双方互相将对方放进心锁时，锁才会打开，双方才会知晓彼此的心意。

### 2.2 包含的功能模块

- 华为账号快捷登录与手机号授权
- 心锁管理（创建、查看、撤销、删除）
- 双向匹配与解锁仪式
- 邀请裂变海报
- 个人账户管理（注销）
- Push 通知

### 2.3 不包含的功能范围（V1.0）

- 即时聊天/消息功能
- 社交 feed/动态
- 匹配推荐算法
- 付费功能/内购
- 多语言国际化（首版仅中文）
- iOS/Android 客户端

## 3. Definitions（术语）

| 术语 | 英文 | 定义 |
|---|---|---|
| 心锁 | Heart Lock | 用户将一份喜欢收藏起来，加密存放在服务器的数据容器 |
| 放进心锁 | Lock | 创建心锁的行为，即用户对某人表达喜欢 |
| 解锁 | Unlock | 双方互相喜欢后的匹配解锁行为 |
| 心愿 | Wish | 用户当前正在等待回应的心锁状态 |
| 收藏喜欢 | Collect Like | 内部对"创建心锁"的行为描述 |
| 打开心锁 | Open Lock | 匹配成功后的仪式 |
| 匹配 | Match | 双方互相将对方放进心锁的事件 |
| 撤回 | Revoke | 取消等待，保留历史记录 |
| 永久删除 | Destroy | 彻底销毁心锁内容 |
| 手机号 Hash | Phone Hash | 对用户手机号进行加盐哈希处理后的标识 |
| 邀请卡片 | Invitation Card | 用于裂变传播的精美海报 |

## 4. Requirements（需求）

### 4.1 用户角色

| 角色 | 描述 |
|---|---|
| 未注册用户 | 首次打开 App，尚未登录的用户 |
| 已注册用户 | 已完成华为账号登录和手机号授权的用户 |
| 匹配用户 | 已成功双向匹配的用户 |

### 4.2 功能性需求

#### 4.2.1 登录与注册

**REQ-001** 用户可以通过华为账号一键登录
**REQ-002** 登录后请求用户授权手机号
**REQ-003** 用户授权后，后端对手机号进行加盐哈希存储
**REQ-004** 未授权手机号的用户无法创建心锁
**REQ-005** 用户可随时退出登录

#### 4.2.2 心锁管理

**REQ-006** 用户可通过输入对方手机号创建心锁
**REQ-007** 创建心锁时需要填写一段文字内容（1-500 字）
**REQ-008** 创建心锁后内容在服务器端加密存储
**REQ-009** 用户可在首页查看自己的心锁列表
**REQ-010** 心锁列表显示状态：等待中 / 已匹配 / 已撤回
**REQ-011** 用户可撤回等待中的心锁
**REQ-012** 用户可永久删除已撤回的心锁
**REQ-013** 首页显示当前心锁数量（如"心锁 1/3"）

#### 4.2.3 匹配与解锁

**REQ-014** 当 B 创建的心锁目标手机号为 A 时，系统自动检测是否形成双向匹配
**REQ-015** 匹配成功后立即触发解锁仪式
**REQ-016** 解锁仪式包含：震动、锁打开动画、对方写给自己的第一句话
**REQ-017** 匹配成功后双方同时收到 Push 通知
**REQ-018** 匹配成功后双方可查看对方写给自己的完整内容

#### 4.2.4 邀请裂变

**REQ-019** 用户创建心锁后可生成一张邀请海报
**REQ-020** 邀请海报不透露任何发送者身份和内容
**REQ-021** 海报包含引导语和"打开心锁"按钮
**REQ-022** 通过海报下载注册的用户，登录后如输入对应手机号即可完成匹配

#### 4.2.5 个人中心

**REQ-023** 用户可查看自己的账户信息
**REQ-024** 用户可注销账户，注销后所有数据彻底删除
**REQ-025** 注销操作需要二次确认

#### 4.2.6 Push 通知

**REQ-026** 匹配成功后立即向双方发送 Push
**REQ-027** 除此之外，系统不发送任何其他类型的通知
**REQ-028** 不允许发送"有人喜欢你"类暗示性通知

### 4.3 非功能性需求

#### 4.3.1 安全

**REQ-029** 所有数据传输使用 TLS 1.3 加密
**REQ-030** 心锁内容在数据库中以 AES-256 加密存储
**REQ-031** 手机号使用加盐哈希（bcrypt + salt）存储
**REQ-032** 注销时所有数据不可恢复删除

#### 4.3.2 性能

**REQ-033** 心锁创建请求响应时间 < 500ms
**REQ-034** 匹配检测延迟 < 2s（从创建到检测完成）
**REQ-035** 解锁仪式动画流畅无卡顿（60fps）

#### 4.3.3 可用性

**REQ-036** App 启动时间 < 3s
**REQ-037** 支持 HarmonyOS NEXT API 12+
**REQ-038** 离线状态下可查看已加载的心锁列表（缓存）

## 5. Business Rules（业务规则）

> **注**：本文档仅列出 PRD 级别的关键业务规则摘要。完整业务规则见 [BusinessRules.md](./BusinessRules.md)。

**RULE-001** 一个人一生只能向同一个手机号创建一次心锁，即使撤回或删除后也不能再次创建
**RULE-002** 一个人最多同时维护 3 个等待中的心锁
**RULE-003** 撤销后改变状态，保留操作记录；永久删除后彻底销毁内容
**RULE-004** 系统不发送任何"暗示性"通知
**RULE-005** 注销账户即永久删除全部数据
**RULE-006** 匹配必须双向，任何单向喜欢永远保密

## 6. Acceptance Criteria（验收标准）

| 编号 | 验收标准 | 关联需求 |
|---|---|---|
| AC-001 | 用户使用华为账号完成登录并授权手机号，整个过程无需手动输入 | REQ-001 ~ REQ-005 |
| AC-002 | 用户输入对方手机号和文字，点击"放进心锁"后成功创建心锁 | REQ-006 ~ REQ-008 |
| AC-003 | 首页准确显示心锁数量和状态 | REQ-009 ~ REQ-013 |
| AC-004 | 当双方互相创建心锁后，在 2 秒内触发解锁仪式 | REQ-014 ~ REQ-018 |
| AC-005 | 邀请海报不包含任何身份信息，下载后可通过海报内容引导新用户 | REQ-019 ~ REQ-022 |
| AC-006 | 注销账户后所有数据不可恢复 | REQ-023 ~ REQ-025 |
| AC-007 | 系统仅发送匹配成功的 Push，无其他推送 | REQ-026 ~ REQ-028 |
