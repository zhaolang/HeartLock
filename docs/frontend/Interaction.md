## 1. Purpose（目的）

定义 HeartLock（心锁）的核心交互规范和动效标准，确保用户交互体验的一致性和高品质。

---

## 2. Scope（范围）

涵盖手势操作、动效规范、反馈机制和异常状态处理。

---

## 3. Gestures（手势操作）

| 操作 | 区域 | 响应 |
|---|---|---|
| 下拉 | 首页列表 | 刷新心锁列表 |
| 点击 | 心锁卡片 | 进入心锁详情 |
| 上滑 | 详情页 | 返回列表 |
| 长按 | 心锁卡片（WAITING）| 弹出快捷操作菜单（撤回/生成卡片）|
| 左滑 | 心锁卡片（WAITING）| 显示「撤回」快捷按钮 |

---

## 4. Animations（动效规范）

### 4.1 通用动效

| 场景 | 动效 | 时长 | 缓动函数 |
|---|---|---|---|
| 页面切换 | 淡入淡出 | 300ms | ease-in-out |
| 按钮点击 | 透明度 0.8 → 1.0 | 150ms | ease-out |
| 列表加载 | 逐项淡入（stagger 50ms）| 300ms/item | ease-out |
| 弹窗出现 | 缩放 0.95 → 1.0 + 背景遮罩淡入 | 250ms | spring(0.4, 1.0) |
| Toast 提示 | 底部滑入，3s 后滑出 | 300ms in / 300ms out | ease-in-out |

### 4.2 特色动效：解锁仪式

解锁仪式是 HeartLock 最重要的交互时刻。详细时序见 [UISpec.md](./UISpec.md#46-匹配解锁仪式页)。

### 4.3 心锁创建动效

1. 点击「放进心锁」按钮 → 按钮进入 loading 状态（旋转 1.5s）
2. 结果弹出：
   - 匹配成功 → 直接跳转解锁仪式
   - 未匹配 → 心锁图标快速闭合动画 + "已收藏" Toast

---

## 5. Feedback（反馈机制）

### 5.1 触觉反馈

| 场景 | 反馈类型 |
|---|---|
| 点击核心按钮（放进心锁）| Light Impact |
| 解锁仪式触发 | Strong Impact + 持续震动 |
| 撤回/删除确认 | Warning Impact |

### 5.2 声音反馈（可选，V1.0 暂不实现）

| 场景 | 声音 |
|---|---|
| 创建心锁 | 锁闭合声 |
| 解锁仪式 | 锁打开声 + 和弦提示音 |

### 5.3 视觉反馈

| 场景 | 反馈 |
|---|---|
| 操作成功 | 绿色 Toast + 图标动画 |
| 操作失败 | 红色 Toast + 错误原因 |
| 网络异常 | 页面内提示条"当前网络不可用" |
| 数据加载 | 骨架屏（Skeleton Loading）|

---

## 6. Error & Empty States（异常与空状态）

### 6.1 空状态

| 页面 | 空状态文案 | 图标 |
|---|---|---|
| 心锁列表 | "尚未收藏任何喜欢" + "第一份喜欢，值得好好想想" | 🔒 |
| 已解锁列表 | "还没有互相喜欢的人" | 💛 |
| 邀请卡片 | "尚未生成邀请卡片" | 📄 |

### 6.2 错误状态

| 场景 | 提示 | 操作建议 |
|---|---|---|
| 网络不可用 | "当前网络不可用" | "重试" 按钮 |
| 创建心锁失败 - 已达上限 | "心锁已达上限（3/3），需要先撤回一个" | "去管理" 按钮 |
| 创建心锁失败 - 已存在 | "已经收藏过这份喜欢了" | "知道了" 按钮 |
| 创建心锁失败 - 不能给自己 | "心锁不能留给自己哦" | 返回编辑 |
| Token 过期 | "登录已过期，请重新登录" | 自动跳转登录页 |
